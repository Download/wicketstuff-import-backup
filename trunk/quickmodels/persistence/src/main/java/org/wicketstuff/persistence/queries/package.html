This package contains the API for complex queries of persistent objects. The primary interface for a complex query is the QueryElement interface. It is passed to FacadeFactory.forComplexQuery() or similar methods on PersistenceUtils (or if you are using the Quick Models library for Wicket, you pass it to Queries.COMPLEX.builder([type]).setObject()).

FieldElementQuery lets you specify a value to match against for a specific field on a specific type. The and() and or() methods let you combine these into compound queries - represented by the class CompoundQueryElement. You can also create trees of queries (think of CompoundQueryElement as a set of parentheses) using CompoundQueryElement.create().
